Multi Channel Temperature Measurement System (mctms)
====================================================


mctms block-level architecture:
- the mctms shall be using the MQTT protocol for communication

- MQTT demands, that a system consists of one broker and one or several clients communicating via the broker.
  Therefore, the system architecture of mctms is as follows:
  
- a broker (mosquitto), running on a raspberry pi (rpi0)

- a recording client to collect the temperature measurement data, add a timestamp and store the data to the file system.
  The recording client is running on a windows systenm (win0), it is named mqtt_recording_client.py, a first implemention is done

- one or several temperature measurement clients (tmc0 to tmcx)
  - each of the temperature measurement clients are implemented on a esp8266 single-board controller.
  - the clients are named tmc0, tmc1 and so on, where tmc is short for "temperature measurement client"
  - each temperature measurement client is connected to up to 16 temperature sensors organized in 2 onewire-busses
  - a max. number of eight sensors per onewire-bus are called a sensorbank
  - sb0 means sensorbank0 and sb1 is the name for the second sensorbank
  - the temperature measurement results shall get published via MQTT with one topic per sensorbank
  - this means, we have two topics per client. e.g. for the first client there are he topics tmc0/sb0 and tmc1/sb1
  
- a command and control client (optional)
